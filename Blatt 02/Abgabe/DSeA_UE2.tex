\section*{Nr.2}

Wir verwenden im folgenden Mergesort zum sortieren des Array. Nach dem Hinweis gilt für die Konstanten: c = 1:
\newline
\newline
\underline{Mergesort:} \[T_M(n) = n^{\log_2(2)} + n \log_2(n) = n + n \log_2(n) \]
\underline{Binäresuche:}  \[T_B(n) = n^{\log_2(1)} + n^{0} \log_2{n}  = 1 + \log_2{n}\]
\newline
$\Rightarrow$ \underline{sortierte Suche:} \[T_1(n) = T_M(n) + T_B(n)\cdot k  \;\;\;\;  // k\;Iterationen\;der\;Schleife \]
\newline
\underline{lineare Suche:}   \[T_2(n) = n \cdot k\]
\newline
\newline
Wir suchen also jenes k, so dass gilt:
\[T_1(n) - T_2(n) = 0\]
\[\Leftrightarrow n(1+\log_2(n)) + k_{grenz}(1+\log_2(n)) - n\cdot k_{grenz} = 0\]
\[\Leftrightarrow k_{grenz} = \frac{-n(1+\log_2(n))}{1+\log_2(n)-n} \;\;\;\;// bei\;n > 2 \]
\newline 
\newline
Für alle $k > k_{grenz}$ macht es Sinn den Array zuerst zu sortieren und anschließend zu suchen.
Für $n < 2$ oder $k < k_{grenz}$ ist die lineare Suche effizienter.

\section*{Nr.3}
\subsection*{a)}
$Beh: 2^{n+k} \in O(n^{2)}$
\newline
$Bew: $
\[\limesinf \frac{2^{n+k}}{2^n} = \limesinf \frac{2^{n} \cdot 2^{k}}{2^n} = \limesinf 2^{k} = 2^{k} < \infty \]
\newline 
$q.e.d$
\newline

\subsection*{b)}
$Beh: 2^{2n} \notin O(n^{2)}$
\newline
$Bew: $
\[\limesinf \frac{2^{2n}}{2^n} = \limesinf 2^{2n} \cdot 2^{-n} = \limesinf 2^{2n-n} = \limesinf 2^{n}  = \infty \not< \infty\]
\newline 
$q.e.d$


\subsection*{c)}
$Beh: \log(n!) \in O(n \cdot \log(n))$
\newline
$Bew: $
\[Beachte: \forall m<n~~ m,n\in\mathbb{N} | \log(n)>\log(m)\]
\[Beachte:  \log(n!) = \sum^{n}_{k=1} log(k) \le \sum^{n}_{k=1} log(n) = n \cdot \log(n)\]
\[\Rightarrow \limesinf \frac{\log(n!)}{n \cdot \log(n)} \le  \limesinf \frac{n\log(n)}{n\log(n)} = 1 < \infty\]
\newline 
$q.e.d$


\subsection*{d)}
$Beh: 2^{kn} \notin O(2^{n})$
\newline
$Bew: $ 
\[\limesinf \frac{2^{kn}}{2^{n}} =  \limesinf 2^{(k-1)\cdot n} = \infty \not< \infty\]
$q.e.d$

\subsection*{e)}
$Beh: 2^{\frac{n}{2}} \in O(\frac{2^{n}}{log(n)})$
\newline
$Bew: $
\[\limesinf \frac{2^{\frac{n}{2}}}{\frac{2^{n}}{log(n)}}=  \limesinf \frac{2^{\frac{n}{2}} \log(n)}{2^{n}} = \limesinf \frac{\log(n)}{2^{\frac{n}{2}}}  = 0 < \infty\]
\newline 
$q.e.d$

\section*{Nr. 4}
\subsection*{a)}
\underline{Zeile 1-5:} 
Kopiere die erste sortierte Hälfte des Arrays A in ein Hilfsarray B.

\underline{Zeile 6-13:} 
Vergleiche das Hilfsarray mit der zweiten Hälfte des Arrays A und sortiere die Zahlen in der richtigen Reihenfolge in das Array A.

\underline{Zeile 14-16:} 
Falls nach dem Vergleichen Elemente in B verblieben sind, kopiere diese in der gleichen Reihenfolge ans Ende von A.
\newline
Alle Elemente im Array A, welche zwischen k und j stehen, können überschrieben werden, weil diese Elemente entweder im Array B stehen oder bereits in die erste Hälfte des Arrays A kopiert wurden. Alle Elemente zwischen left und k sind bereits sortiert. (Für Sortierung: siehe b))
\subsection*{b)}
Wie in a) geklärt, gehen bei dem Algorithmus keine Daten verloren. Nach jedem Durchlauf der zweiten Schleife sind die Elemente zwischen left und k sortiert, da nach dem sort Schritt die linke Hälfte und die rechte Hälfte vorsoriert sind. Solleten alle Elemente der linken Hälfte kleiner als das größte Element der rechten Hälfte sein, wird nur die zweite Schleife durchlaufen, sonst werden die restlichen Elemente im Array A mit den noch nicht kopierten Elementen des Array B überschrieben.
\subsection*{c)}
\subsubsection*{Speicherbedarf:}

Im Gegensatz zur merge Funktion aus der Vorlesung, welche ein Hilfsarray der Größe $|A|$ benötigt, genügt hier ein Hilfsarray B der Größe $\frac{|A|}{2}$, da das Array A nur zur Hälfte und nicht vollständig kopiert werden muss.

\subsubsection*{Kopieroperationen:}
\paragraph{Vorlesung}
Das komplette Array A der Größe $|A|$ wird sortiert nach B kopiert, anschließend wird das Array B nach A kopiert $\Rightarrow$ $2\cdot|A|$ Kopiervorgänge.
\subparagraph{Best Case}
Alle Elemente der linken Hälfte sind kleiner als das kleinste Element der rechten Hälfte $\Rightarrow$ $\frac{|A|}{2}$ Kopiervorgänge und das Kopieren von A in B sind $\frac{|A|}{2}$ Kopiervorgänge $\Rightarrow$ $|A|$ Kopiervorgänge.
\subparagraph{Worst Case}
Das kleinste Element der linken Hälfte ist größer, als das größte Element der rechten Hälfte $\Rightarrow$ ${|A|}$ Kopiervorgänge und das Kopieren von A in B sind $\frac{|A|}{2}$ Kopiervorgänge $\Rightarrow$ $1,5\cdot|A|$ Kopiervorgänge.

\subsubsection*{Elementvergleiche:}
\subparagraph{Best Case}
Alle Elemente der linken Hälfte sind kleiner als das kleinste Element der rechten Hälfte oder das kleinste Element der linken Hälfte ist größer, als das größte Element der rechten Hälfte$\Rightarrow$ $\frac{|A|}{2}$ Elementarvergleiche.
\subparagraph{Worst Case}
Das kleinste Element der rechten Hälfte ist kleiner, als das kleinste Element der rechten Hälfte und das größte Element der linken Hälfte ist kleiner als das größte Element der rechten Hälfte. $\Rightarrow$ $|A|$ Elementarvergleiche.

Beide merge Funktionen sind bei den Elementarvergleichen identisch.
